# Smiley v12.0.9

**Patch release** — custom GIF add/delete/apply reliability.

## Fixes

### 1. Cannot add or apply custom GIF activities
- **Root cause:** `add_custom` saved through `save_config`, which always tried to re-resolve the **idle GIF** URL on every custom save. If your idle GIF was an old Tenor page link or failed network resolution, adding a custom activity failed even when the new activity GIF was fine.
- **Fix:** `add_custom` and `remove_custom` now persist via a direct `persist_config` path. `save_config` only resolves idle GIF when you actually change it in Settings → Auto.

### 2. Cannot delete custom activities (Terminal skin)
- **Root cause:** Terminal skin `.row-wrap` lacked `position: relative`, so the delete (×) button was absolutely positioned against the wrong ancestor and could land off-screen or be unclickable.
- **Fix:** Terminal rows use flex layout with a static delete button; delete controls are always visible on custom tiles.

### 3. Delete / favorite clicks could hit the activity tile
- **Fix:** `stopPropagation` + `preventDefault` on favorite and delete actions so picking and deleting don't race.

## Verification

```bash
cd Smiley.v12 && npm run build
cd Smiley.v12/src-tauri && cargo test && cargo check
```

### Custom GIF add + apply
1. Settings → Auto — leave idle GIF as-is (or set a Tenor page link).
2. **＋** → New activity → paste Tenor URL → **Test** → **Add**.
3. **My Activities** → tap the tile → Discord shows your custom details/GIF.

### Custom delete
1. **My Activities** → click **×** on a custom tile (visible without hover).
2. Confirm — tile disappears; restart app — still gone.
3. Terminal skin: `./new` → create one → delete via **×** on the row.
