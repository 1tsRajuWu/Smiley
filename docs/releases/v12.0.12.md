# Smiley v12.0.12

**Patch release** — fixes install database heartbeats that were silently failing on every launch.

## Root cause

Packaged v12 builds **did** load `downloads.registry.json` and POST to Supabase, but the `installs` trigger called `digest()` without the Supabase `extensions` schema on `search_path`. Every insert/update failed with:

`function digest(text, unknown) does not exist`

PostgREST surfaced that as HTTP 404, so the dashboard kept showing old v5-era rows while new launches wrote nothing. Section telemetry never synced either (`INSTALL_REGISTERED` stayed false).

## What's fixed

- **DB trigger** — `extensions.digest(...)` + `search_path = public, extensions` (`scripts/migrate-fix-digest-trigger.sql`)
- **Launch count** — geo enrichment patches no longer inflate `launch_count`; heartbeats send `client_heartbeat_at`
- **Richer install row** — `runtime_kind`, `host_os_name`, `app_channel`, music/game/coding last-seen fields + enable flags
- **Stats dashboard** — runtime/channel charts and expanded install detail columns
- **Release CI** — fails if Supabase registry secrets are missing (no more empty registry bundles)
- **Auth headers** — always send `Authorization: Bearer` for JWT and `sb_publishable_` anon keys

## How to verify

1. Install **v12.0.12** and launch once.
2. Log: `tail -f ~/Library/Application\ Support/Smiley/logs/smiley.log` — expect `registry: install heartbeat ok` (not HTTP 404 / digest).
3. Stats dashboard / Supabase `installs` — new or updated row with `app_version` `12.0.12`, `runtime_kind` `tauri`.
