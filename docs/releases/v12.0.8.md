# Smiley v12.0.8

**Patch release** — custom GIF URL resolution rebuilt from scratch.

## Fixes

### Custom GIF URLs not working
- **Root cause:** Tenor page links no longer embed direct `media.tenor.com` URLs in view-page HTML (SPA shell). The old scraper failed when users pasted Tenor page or short links. Tenor’s public API is also discontinued.
- **Fix:** New `tenor.rs` resolver — extracts post ID from any Tenor URL, fetches the lightweight `/embed/{id}` page, parses `gif-json` `media_formats`, and normalizes CDN URLs. Fallbacks: HTML scrape, redirect following for short links, paste cleanup (`<https://…>` wrappers).

### Custom GIF UI rebuilt
- **Test** button + live preview before save (create dialog + Settings → Auto → Idle GIF).
- Form resets when opening **New activity**.
- **My Activities** shows a create tile when the list is empty.
- Resolved URL displayed under preview so users know what Discord will use.

## Verification

```bash
cd Smiley.v12 && npm run build
cd Smiley.v12/src-tauri && cargo test && cargo check
```

### Custom GIF URL
1. **＋** → New activity → paste a Tenor page link (e.g. `https://tenor.com/view/…-gif-5336135`).
2. Click **Test** — preview should appear with a `media.tenor.com` URL.
3. **Add** → tile shows GIF → select → Discord shows your custom details/GIF.

### Idle GIF
1. Settings → Auto → paste Tenor URL → **Test** → **Save**.
