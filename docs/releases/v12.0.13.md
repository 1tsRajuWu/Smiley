# Smiley v12.0.13 — Valorant TDM / DM Discord scores

**Patch release** — Team Deathmatch and Deathmatch scores sync to Discord Rich Presence again (and the in-app live state line).

## Root cause

After the v8/v12 scoreboard merge, Smiley preferred the local `/scoreboard` payload for team scores. That payload often carries spike-style `RoundScore: 0` for TDM (no `NumPoints`), which produced a sticky `0-0` and blocked chat `partyOwnerMatchScore*` fallback. FFA Deathmatch could also pick up leftover ally–enemy chat scores, and a `?` in the TDM kills-sum loop aborted on the first player without stats.

## What's fixed

- **TDM** — read `Teams[].NumPoints` from the core-game match payload; ignore scoreboard `RoundScore` noise; fall back to summed team kills or chat ally–enemy points
- **DM** — kills-only (`N kills`); never use chat team scores
- **Recovery** — prefer chat `queueId` for mode classification so a misread local ModeID cannot keep a fake DM kills line over real TDM points
- **Parsing** — coerce Riot JSON ints / floats / numeric strings for kills and scores
- **Comp / Unrated / Swiftplay** — still use `RoundScore` / rounds as before

## How to verify

1. Install **v12.0.13**, open Discord + Smiley, enable **Valorant presence**.
2. Launch Valorant (Riot Client lockfile present).
3. **Team Deathmatch** — Discord state should show live points like `71-55 · … · Team Deathmatch` (not `0-0`, not `N kills`).
4. **Deathmatch** — Discord state should show `14 kills · … · Deathmatch` (not ally–enemy `3-2`).
5. Spot-check **Competitive / Unrated** — round score still updates as before.

## Privacy / scope

- Local Riot lockfile + `127.0.0.1` APIs only
- No inject, no memory reads, no Tracker Network
