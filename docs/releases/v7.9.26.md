## Smiley v7.9.26 — Faster gaming presence updates

**Release date: July 8, 2026**

Gaming presence felt one phase behind (lobby → queue → agent select → match) because throttles still delayed Discord and UI pushes even when game state changed. This patch removes that lag while keeping safe rate limits for steady-state polls.

### Changes

- **Immediate on transition** — any session/phase change bypasses Discord and renderer throttles (`game-sync.js`)
- **Faster polls** — foreground game detection: macOS 8s→5s, Windows 5s→3.5s (`now-gaming.js`)
- **Faster Riot API cadence** — lobby 5s→3s, queue/agent-select 2s, in-match 4s→2.5s (`game-providers/index.js`)
- **Reduced steady-state throttles** — Discord 5s→2.5s (2s in-match), UI preview 1.5s→600ms; throttles only apply when state is unchanged
- **Richer dedup sig** — `applyGamePresence` now keys on `phase`, queue/pregame/lobby flags, and `scoreHint` so phase transitions always reach Discord

### Kept (unchanged)

- `SMILEY_DISABLE_NOW_GAMING=1` kill switch
- Background poll intervals (tray/wallpaper pause)
- macOS JXA backoff on failures
- 2s identical-payload dedup in `applyGamePresence` (same sig only)
- Music/coding sync throttles (unchanged)

### Quality

- `npm run game-presence-check` (×2) — 205 checks
- `npm run security-check` — 21 checks
