# Smiley v8.0.6

**Instant music presence** — track changes show up within ~2 seconds instead of the old ~24s throttle.

## Changes

- **Music poll interval:** Dedicated background thread probes Spotify / Apple Music every **2s** while "Listening to music" is active and Discord is connected (was ~24s: 8s live tick × every 3rd tick).
- **Track-change dedup:** Discord RPC only updates when title + artist + app + playing state changes — no spam on identical polls.
- **Low CPU when idle:** Music thread sleeps 8s when not on the listening activity; gaming/Valorant live tick stays at 8s on its own thread.
- **UI never blocked:** osascript probes run on worker threads with a 2.2s hard timeout (unchanged).
