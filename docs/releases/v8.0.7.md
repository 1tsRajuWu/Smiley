# Smiley v8.0.7

**In-app updates + instant music presence** — download and install updates inside the app (like v7), plus faster music track sync.

## New

- **In-app updater (Tauri):** Settings → App → **Check for updates** checks `latest.json` on GitHub Releases, downloads a signed update bundle, then **Restart to update**. Silent background check ~15s after launch (auto-download; restart is manual).
- **Music poll interval:** Dedicated background thread probes Spotify / Apple Music every **2s** while "Listening to music" is active and Discord is connected (was ~24s).
- **Track-change dedup:** Discord RPC only updates when title + artist + app + playing state changes.
- **Low CPU when idle:** Music thread sleeps 8s when not on the listening activity; gaming/Valorant live tick stays at 8s.

## Notes

- **First in-app update path:** Users on v8.0.5 or older must install **v8.0.7 once** from GitHub (DMG/Setup). After that, future updates can install in-app.
- **macOS:** Ad-hoc signed builds (no Apple Developer ID). Gatekeeper may still warn on first open after update — same as v7. Install to `/Applications` before updating.
