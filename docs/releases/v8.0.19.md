# Smiley v8.0.19 — Gaming presence + security pass

Patch release fixing CS2 and other non-Valorant game detection, plus a v8 security hardening sweep.

## Gaming fixes
- **CS2 and Steam games** — process probe now matches v7 known-game catalog (CS2, Dota 2, Apex, etc.) with correct titles, `Playing` state, and Steam capsule artwork on Discord.
- **macOS CS2** — detects `Counter-Strike 2` and `cs2` process names (not just Windows `cs2.exe`).
- **Stale Valorant lobby** — foreground non-Riot games (e.g. CS2) now beat an idle Valorant/LoL lobby instead of leaving Discord stuck on Riot.
- **Live gaming default** — process probe runs when **Live gaming** is on (no longer requires the separate probe toggle).
- **Idle League client** — open Riot client no longer hijacks Discord when you're playing another game.
- **Faster live-game polls** — 2s cadence for any `live-*` game session (CS2, etc.), not only Valorant.

## Security fixes
- **Log redaction** — bearer tokens, auth headers, and long secret-like strings are stripped from UI log lines.
- **Discord button URL allowlist** — RPC button links restricted to GitHub releases only (config sanitize).
- **Settings XSS** — category dropdown options now HTML-escaped.
- **CSP** — Steam CDN hosts added to `img-src` for capsule artwork in the webview.

## Verification
- `cargo test` (42 tests)
- `cargo check`
- `npm run build`

## Privacy / scope
- Process list probe only — no inject, no memory reads
- Riot lockfile + `127.0.0.1` local APIs only
- Open URL allowlist unchanged: PayPal donate + GitHub releases/updater
