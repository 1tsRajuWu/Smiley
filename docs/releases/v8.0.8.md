# Smiley v8.0.8

**Music GIF presence + live coding detection**

## Fixed

- **Music banner GIF on Discord:** Listening presence always uses the activity's **animated Tenor GIF** as `large_image` (never static album art or the app icon). URLs are sanitized and sent as external RPC images; activity type is **Listening**.
- **Coding stuck on idle:** Coding no longer stays on the static "Building something cool" line when an editor is open — live window-title parsing mirrors v7 `now-coding.js`.

## New

- **Live coding sync:** While **Coding** is selected, Smiley detects Cursor, VS Code, Xcode, JetBrains IDEs, terminals, and more via macOS foreground osascript. Discord shows the editor name and live line (e.g. `Editing app.rs · Project Smiley`). Settings → Discord → **Live coding**.

## Notes

- **Coding detection:** macOS only for now; uses System Events (same as v7). When Smiley/Discord is frontmost, falls back to a running editor.
- **Music GIF:** Tenor URLs only — same trusted hosts as v7 Discord RPC.
