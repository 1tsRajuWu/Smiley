# Smiley v12.0.6

**Patch release** — custom GIF URLs, music presence priority, and Tenor browse links.

## Fixes

### 1. Custom GIF / URL not working
- **Root cause:** Only direct `media.tenor.com` URLs were accepted; Tenor page links were rejected or silently replaced with a fallback GIF. Invalid URLs in custom activities were swapped for a default without telling the user.
- **Fix:** `resolve_gif_url` resolves Tenor page links to CDN URLs (HTTP fetch + HTML parse). Custom activity save and idle GIF settings validate/resolve on save and return clear errors instead of silent fallback.

### 2. Music overtaking other activities
- **Root cause:** The background music thread could race with manual activity picks — it cloned the listening slot state, then called `discord.set()` after the user had already selected gaming/coding/custom, overwriting Discord presence.
- **Fix:** `listening_slot_active()` guard re-checked immediately before every music `discord.set()` and status write. Music only drives presence when **Listening** is explicitly selected.

### 3. GIF source browse links
- **Added:** Tenor browse button in **New activity** dialog and **Settings → Auto → Idle GIF** so users can open Tenor, copy a link, and paste it back (direct CDN or page URL).

### Custom GIF
1. **＋ New activity** → paste a `tenor.com/view/…` page URL → **Add**.
2. Pick the custom activity — Discord should show your GIF (not the generic streaming fallback).
3. Settings → Auto → Idle GIF → paste Tenor page URL → **Save** → **Idle** — quiet-hours idle uses your GIF.

### Music priority
1. Connect → select **Listening** → play music (Discord shows track).
2. Select **Gaming** or a custom GIF while music keeps playing.
3. Discord should switch to the picked activity and stay there (music must not steal it back).

### Tenor browse link
1. **＋ New activity** → click **Tenor** → browser opens tenor.com.
2. Copy a GIF link from Tenor → paste into GIF URL field → save.
