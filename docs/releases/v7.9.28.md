## Smiley v7.9.28 — Clear stale gaming presence on force-quit

**Release date: July 8, 2026**

### Force-quit / direct shutdown still showing "Playing" (user report)

Discord could stay on a live game (CS2 capsule, Valorant match, etc.) after force-quitting the game or closing Smiley abruptly.

- **Process-gone = immediate clear** — sticky gaming session drops as soon as the game executable leaves the process list (alt-tab is unchanged; the process stays alive)
- **Riot chat no longer trusted without game process** — Valorant/LoL lockfile presence is ignored when `VALORANT-Win64-Shipping` / League client is not running (Riot Client alone does not count)
- **Faster active-game polls** — ~2s while a game is tracked, ~2.5–3s idle (was up to 5s on macOS)
- **Idle gaming template strips live artwork** — reverting to static Gaming clears CS2/Valorant large/small images from the RPC payload
- **Quit cleanup** — `before-quit` calls `clearActivity()` before destroying the RPC client; game-sync resets presence dedup on activity switch

### Grace period

No multi-minute sticky TTL. Alt-tab keeps the game process running, so presence stays. Force-quit clears on the next foreground poll (≤2s when a game was active).
