## Smiley v7.9.25 — Valorant TDM score on Discord

**Release date: July 8, 2026**

Team Deathmatch scores were missing on Discord because ModeID `TeamDeathmatch` was misclassified as FFA Deathmatch (substring match), which dropped chat ally–enemy points. Presence stays **local-only** (Riot lockfile + local APIs) — no Tracker.gg, no injection, no malware.

### Fixes

- **TDM score** — classify `TeamDeathmatch` / `hurm` before FFA; parse `Teams[].NumPoints` (ally-first via `TeamID`); fall back to summed team kills or chat `partyOwnerMatchScore*`.
- **Deathmatch (FFA)** — still kills-only (`N kills`); never fake round scores.
- **Spike / Comp / Unrated / Swiftplay** — still `RoundScore` ally–enemy.
- **Presence** — ValShy-inspired richness: agent · map · score · party · mode (Smiley branding, `showScore` respected).

### Example Discord payloads

**Team Deathmatch**
```text
details:  Jett · Kasbah
state:    71-55 · 2-Stack · Team Deathmatch
```

**Deathmatch**
```text
details:  Jett · Ascent
state:    14 kills · Solo · Deathmatch
```

**Competitive**
```text
details:  Jett · Ascent
state:    8-6 · 5-Stack · Competitive · 12/9/4
```

### Security

- Local Riot Client lockfile + HTTPS to `127.0.0.1` only for live match data
- No Tracker.gg / third-party competitive trackers
- No memory injection, overlays, or spyware
- Optional encrypted Riot API key for rank (unchanged)

### Quality

- `npm run game-presence-check` (×2) — 205 checks including TDM NumPoints / DM kills / Comp RoundScore fixtures
- `npm run security-check`
- `node --check` on Valorant presence entrypoints
