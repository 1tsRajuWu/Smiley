## Smiley v7.8.2 — Gaming sync bug fixes & security hardening

**Release date: July 8, 2026**

Made by [@1tsRaj](https://github.com/1tsRaj).

### Gaming sync bug fixes

- **Riot session preserved** — Valorant/LoL lockfile data no longer overridden by generic foreground window titles
- **Valorant party size** — parse `partySize` / `partyMembers` correctly; stop treating `partySizeMax` as current party size
- **Stale presence cleared** — Discord resets to generic Gaming when no live session is detected
- **Match poll interval** — Riot in-match polls use 4s timing (was stuck at menu rate)
- **Cover art toggle** — changing **Use game cover art** immediately refreshes presence and UI preview
- **Settings sync** — gaming toggles persist and apply in the renderer after save

### Security audit pass

- **Gaming sync verified clean** — no malware, injection, or secret exfiltration in Riot/game sync code
- **Defense-in-depth** — explicit deny-list for sensitive game session fields (PUUID, lockfile password, tokens, Riot IDs)
- **Riot client hardening** — removed unused `gameName`/`gameTag` from local presence reads
- **Docs** — `SECURITY.md` gaming sync section; install registry `consent_version` doc aligned to `2026-07-08`
- **Self-checks** — expanded `security-check` and `game-presence-check` (25 checks)
