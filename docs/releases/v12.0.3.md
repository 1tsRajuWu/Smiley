# Smiley v12.0.3

**Patch release** — fixes music not syncing to Discord (Apple Music, Spotify, browser players).

## What's fixed

- **MediaRemote stream** — correct mediaremote-adapter path resolution in dev and packaged `.app` builds (`Resources/mediaremote-adapter`)
- **Instant track on listen** — one-shot `get` probe when Listening activity activates and when the stream starts, so Discord updates immediately without waiting for a skip
- **Stream resilience** — logs adapter/stream failures; restarts stream on disconnect; osascript fallback when adapter unavailable
- **Paused tracks** — title/artist still shown when macOS reports `playing: false` (v7 parity)
- **Gaming guard** — live Valorant/game probe still yields to the listening slot (unchanged)

## How MusicPresence works in v12

1. Select **Listening to music** with **Show now playing** enabled in Settings.
2. Background thread starts `/usr/bin/perl mediaremote-adapter.pl stream --no-artwork` (bundled framework).
3. JSON diff lines on stdout → Rust merges state → `music_apply_track` → Discord RPC (`Listening` type).
4. Linux uses MPRIS D-Bus poll; Windows falls back to idle listening template.
