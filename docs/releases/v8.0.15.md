# Smiley v8.0.15 — updater reliability fix

Patch release for the broken v8 in-app updater fallback path.

## Fixes
- **Real updater flow restored**: the v8 UI now treats Tauri updater `check()` results correctly, so `no update` stays `up to date` instead of jumping straight to **Download from GitHub**.
- **Fallback narrowed to hard failures**: GitHub fallback now appears only when GitHub reports a newer v8 release but the in-app updater cannot resolve or install a signed package for the current platform.
- **Safer release rollout**: the v8 release workflow now keeps GitHub releases in draft until every matrix job finishes, which prevents `latest.json` from going public with only partial platform metadata.

## Verification
- `npm run build`
- `cargo check`
- inspected `latest.json` for `v8.0.13` and `v8.0.14`
- inspected GitHub release assets and workflow behavior

## Compatibility note
- Users already on **v8.0.13 Intel macOS / Windows / Linux** should be able to update in-app once `v8.0.15` is published.
- Users on **Apple Silicon** who checked during the incomplete `v8.0.14` rollout may need one manual install if their app only ever saw the partial release metadata.
