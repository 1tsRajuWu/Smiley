# Smiley v12.0.10

**Patch release** — custom GIF add / delete / apply fixed for real.

## Why v12.0.9 still failed

v12.0.9 fixed idle-GIF blocking `save_config` and Terminal delete layout, but three failure paths remained:

1. **Add rejected common Tenor CDN pastes** — browser “Copy image address” often includes `?hh=&ww=` query params. `sanitize_gif_url` required the string to *literally* `ends_with(".gif")`, so those URLs failed resolve/add (v7 accepts `/\.gif(\?|#|$)/`).
2. **Delete clicks still hit the tile** — fav/× sat as siblings over a transformed `.tile`, so hit-testing could apply presence instead of delete.
3. **Apply felt broken** — empty GIF customs were allowed (Discord showed the generic fallback), Enter in the create form did nothing, and add did not auto-apply when connected.

## Fixes

- Strip `?` / `#` from pasted Tenor CDN URLs; accept `.gif` path with query/hash (v7 parity).
- Prefer `mediumgif` / `gif` over `tinygif` when resolving Tenor embed JSON.
- Require a Tenor GIF on add (frontend + Rust); clear error if missing.
- Create form: Enter / Add submit saves; auto-applies the new activity when Discord is connected.
- Tile fav/delete live in a `pointer-events` overlay so × always wins over pick.
- Larger always-visible delete control; favorites toggle uses `persist_config` (no idle re-resolve).
- Failed Discord presence push rolls back the activity slot (no stranded selection).
- Electron `release.yml` skips `v12.*` tags (v12 builds only via `release-v12.yml`).


### Custom GIF add + apply
1. Connect Discord.
2. **＋** → details + paste Tenor page or `media.tenor.com/….gif` (query params OK) → **Test** → **Add** (or Enter).
3. Discord should update immediately with your custom details/GIF.
4. **My Activities** → tap the tile again anytime to re-apply.

### Custom delete
1. **My Activities** → click **×** (top-left, always visible) → confirm.
2. Tile disappears; restart app — still gone.
3. Terminal skin: `./new` → create → delete via **×** on the row.

## Limitations

- Giphy / non-Tenor hosts are not supported.
