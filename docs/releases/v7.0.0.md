## Smiley v7.0.0 — Final release

**The one and only final version** — Aurora UI, Mac fixes, instant music sync, and low-end performance polish. Everything in one build.

Made by [@1tsRaj](https://github.com/1tsRaj).

> Intermediate tags (v6.x, v7.0.1–v7.0.6, v7.7.7) were development iterations. **v7.0.0** is the canonical final release — it includes all performance optimizations from the final engineering pass.

### Aurora UI (v3)

- **Picker-first layout** — preview dock on the right, list-style activities, circular character preview, underline category rail
- Default for new installs; v1/v2 still available in Settings
- Cyan-violet Aurora Trails theme with light streaks
- Reduced backdrop blur in v3 for low-end PCs

### Mac update fix

- In-app zip downloads no longer stuck at 0% (missing updater helpers restored; stall detection covers 0%)
- HTTP zip fallback resets stall timer during transfer

### Instant music sync

- **macOS** — `mediaremote-adapter` event stream for instant track changes (Music Presence approach); JXA poll fallback
- **Windows/Linux** — native `node-nowplaying` events
- **In-app music preview bar** — album art, title, artist, and smooth progress bar while Listening is active
- **Instant Discord song updates** — track title/artist push immediately; album art follows async
- No stale overwrites — listening activity no longer resets Discord to generic template
- No Discord green elapsed timer for music presence

### Performance & low-end polish

- **Less CPU when tray-hidden** — music sync pauses UI/renderer updates and slows polling while minimized; Discord still updates on song changes
- **Smarter now-playing** — macOS polls less often; mediaremote stream events debounced; progress-only updates throttled
- **Fewer Discord RPC floods** — duplicate presence payloads deduped; progress-only RPC spaced out; tray/status broadcasts debounced
- **Lighter renderer** — GIFs pause when hidden; music progress bar uses rAF batching at 2s intervals; animations freeze in background
- Slower now-playing polls, debounced activity grid renders, lazy GIF picker thumbs
- **Smaller desktop build** — excludes mobile, native C# project, legacy `styles.css`, docs, and scripts from packaged app

### Windows packaging

- `mediaremote-adapter` is macOS-only in `extraResources`; no symlink errors on Windows NSIS
