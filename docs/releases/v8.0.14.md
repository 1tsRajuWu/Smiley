# Smiley v8.0.14 — live board + timer fixes

Patch release for the v8.0.13 live gaming regressions.

## Fixes
- **Valorant live board roster**: preserves ally/enemy team hints from local pregame and core-game payloads, so ally rows no longer fall into the enemy column when Riot omits `TeamID` on the flat player list.
- **Pregame enemy fallback**: reads `EnemyTeam` whether Riot returns an array or nested object, then dedupes merged roster sources before sending them to the app.
- **Core-game roster fallback**: backfills missing team IDs from `Teams[].Players` when the top-level `Players[]` list is present but under-labeled.
- **Discord elapsed timer**: keeps a stable start timestamp across harmless live presence refreshes and only restarts when the live session actually changes phase/map/queue or switches to another game.

## Verification
- `cargo test`
- `cargo check`

## Remaining local API limitation
- Before full lock-in, the local Riot pregame APIs may still omit or delay enemy roster details. When that happens, Smiley now keeps the ally roster correct and only shows enemy rows once the local client actually exposes them.
