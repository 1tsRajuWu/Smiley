# Smiley v12.0.5

**Patch release** ‚Äî restores install database telemetry and improves music sync diagnostics.

## What "database" meant

Users reported *"the database is not taking and sending data"*. In Smiley this refers to the **Supabase install database** (mandatory usage telemetry in [INSTALL-DATABASE.md](../INSTALL-DATABASE.md)) ‚Äî **not** SQLite. v12 shipped without porting v7's `install-registry.js` / `install-telemetry.js`, and the v12 release workflow never bundled `downloads.registry.json`, so **official v12.0.0‚Äì12.0.4 builds sent zero heartbeats** and no `music_sync` / `game_sync` / `coding_sync` section rows.

## Root cause

| Gap | Effect |
|-----|--------|
| No Rust install registry module in v12 | No launch heartbeat, no section telemetry |
| `release-v12.yml` missing Supabase bundle step | Packaged `.app` had no `downloads.registry.json` |
| Telemetry not recorded on music/game/coding events | Dashboard showed stale / empty section data even if installs row existed |

Music Discord sync fixes in v12.0.3‚Äì12.0.4 were separate; this release also adds **log lines when music track data is dropped** (not connected, wrong activity, etc.).

## What's fixed

- **Install registry (Rust)** ‚Äî port of v7 heartbeat + `install_sections` / `install_section_sources` sync
- **Bundled registry** ‚Äî CI writes `downloads.registry.json` into the v12 app Resources on release builds
- **Telemetry hooks** ‚Äî launch, activity select, music track apply, Valorant/game probe, coding live
- **Persisted rollup** ‚Äî `installTelemetry` stored in `config.json` (same schema as v7)
- **Music logging** ‚Äî `smiley.log` records why tracks were dropped before Discord push

## How to verify

### Install database (author dashboard)

1. Install **v12.0.5** from GitHub Releases.
2. Launch Smiley once (packaged build with registry bundled).
3. Check Supabase `installs` ‚Äî new row or updated `last_seen_at`.
4. Select **Listening**, play Apple Music ‚Äî `install_sections.music_sync` should update within ~3s.
5. Log: `tail -f ~/Library/Application\ Support/Smiley/logs/smiley.log` ‚Äî expect `registry: install heartbeat ok` or `registry: section telemetry synced`.

### Music ‚Üí Discord (user)

1. Open Discord desktop ‚Üí Smiley **Connect** ‚Üí **Listening to music**.
2. Settings ‚Üí **Live music** on.
3. Play a track ‚Äî Discord should show title + artist; log should **not** say `track dropped`.
