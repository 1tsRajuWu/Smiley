# Smiley v12.0.11

**Patch release** — custom GIF activities rebuilt from the working v7 model.

## Why customs stayed broken after v12.0.9 / v12.0.10

Built-in tiles worked because they used a simple apply control. Custom tiles layered a full-bleed `pointer-events` overlay plus nested hit targets on top of that path, so apply/delete could miss or race in WKWebView. Delete also relied on `window.confirm`, which can silently no-op in some Tauri webviews. Create used `method="dialog"` submit instead of an explicit save action.

Discord Client ID / RPC payload were **not** the differentiator: customs already used the same `set_activity` → HTTPS Tenor `large_image` path as built-ins. CI still injects `DISCORD_CLIENT_ID` into `src-tauri/discord.app.json` for release builds.

## What was rebuilt

1. **Tile model (v7-faithful)** — outer card owns apply (`data-act=pick`); favorite / delete are sibling buttons only (no full-card overlay).
2. **Click bus** — explicit priority: delete → fav → pick (same order as Electron v7 grid delegation).
3. **Delete** — two-step × confirm (toast “click again”) instead of flaky `window.confirm`.
4. **Create** — explicit `data-act=save-custom` Add button; no `method="dialog"` submit race.
5. **Rust** — stricter add/remove logging, `custom-` id prefix, error if delete id missing; Tenor query-string CDN pastes still resolve.

## Verification

```bash
cd Smiley.v12 && npm run build
cd Smiley.v12/src-tauri && cargo test && cargo check
```

### Manual test

1. Connect Discord.
2. **＋** → details + Tenor page or `media.tenor.com/….gif` (query params OK) → **Test** → **Add**.
3. Discord should update with the custom details/GIF; tile appears under **My Activities**.
4. Tap the custom tile again — presence re-applies (same path as built-ins).
5. Click **×** once → toast asks to confirm → click **×** again → gone; restart app — still gone.
6. Built-in Food/Chill tiles still apply as before.

## Limitations

- No local file picker for custom GIFs (Tenor HTTPS only) — unlike v7’s upload path.
- Giphy / non-Tenor hosts are not supported.
- Delete requires two clicks on × (by design, replaces native confirm).
